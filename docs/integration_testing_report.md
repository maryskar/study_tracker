# Отчет по интеграционному тестированию

## 1. Выполненная работа

Интеграционные тесты переписаны как 10 законченных сценариев взаимодействия модулей `auth`, `database`, `timer` и `api_client`.

Файл сценариев:
- `integration_tests/test_integration_workflows.py`

Вспомогательные фикстуры:
- `integration_tests/conftest.py`

## 2. Перечень сценариев

1. Регистрация, логин, проверка JWT и сохранение Pomodoro-сессии.
2. Повторная регистрация не заменяет существующий аккаунт.
3. Неуспешный логин не создает учебную сессию.
4. Завершенный Pomodoro сохраняет сессию и достижение.
5. Stopwatch обновляет UI-значение и сохраняет длительность.
6. Успешные mock-ответы внешних API используются в пользовательском flow.
7. Timeout внешних API возвращает fallback и не ломает основной flow.
8. Сессии, отчеты и достижения двух пользователей изолированы.
9. Последовательные Pomodoro и Stopwatch очищают scheduler и сохраняются в БД.
10. `stop_timer` без старта безопасен, а short break не выдает Pomodoro-достижение.

## 3. Использованные инструменты

- `pytest`
- `unittest.mock.patch`
- временная SQLite БД на каждый сценарий
- `InlineScheduler` вместо фонового планировщика
- `python-jose` для проверки JWT

## 4. Заглушки и изоляция

Внешние HTTP API не вызываются напрямую. Для `MotivationAPI`, `WorldTimeAPI` и `ScheduleAPI` используются mock-ответы и mock-ошибки.

Планировщик таймера заменен тестовой реализацией `InlineScheduler`, поэтому тесты не зависят от фоновых потоков и проходят стабильно.

## 5. Негативные сценарии

Покрыты:
- повторная регистрация пользователя
- неверный пароль
- timeout внешних API
- остановка таймера без активной сессии
- завершение short break без Pomodoro-достижения

## 6. Результат проверки

Команды запуска:

```bash
python -m pytest integration_tests -m integration -q
python -m pytest
```

Результат:
- интеграционные тесты: `10 passed`
- полный набор тестов: `158 passed`

## 7. CI

Интеграционные тесты запускаются в GitHub Actions job `integration-tests`:

```bash
python -m pytest integration_tests -m integration --junitxml=integration-junit.xml
```

Отчет сохраняется как артефакт `integration-junit.xml`.
