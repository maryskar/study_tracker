# Integration Tests

Интеграционные сценарии проверяют совместную работу модулей `auth`, `database`, `timer` и `api_client`.

## Количество сценариев
- 10 законченных пользовательских кейсов
- позитивные сценарии, негативные сценарии и fallback внешних API
- каждый тест использует изолированную SQLite БД

## Сценарии
1. Регистрация, логин, JWT и сохранение Pomodoro-сессии.
2. Повторная регистрация не заменяет существующий аккаунт.
3. Неуспешный логин не создает учебную сессию.
4. Завершенный Pomodoro сохраняет сессию и достижение.
5. Stopwatch обновляет UI-значение и сохраняет длительность.
6. Успешные mock-ответы внешних API используются в пользовательском flow.
7. Timeout внешних API возвращает fallback и не ломает основной flow.
8. Сессии, отчеты и достижения двух пользователей изолированы.
9. Последовательные Pomodoro и Stopwatch очищают scheduler и сохраняются в БД.
10. `stop_timer` без старта безопасен, а short break не выдает Pomodoro-достижение.

## Локальный запуск
```bash
python -m pytest integration_tests -m integration -v
```

## Особенности
- внешние HTTP API изолируются через `unittest.mock.patch`
- планировщик таймера заменен на `InlineScheduler`
- helper-фикстуры оформляют повторяющиеся шаги регистрации и завершения Pomodoro
