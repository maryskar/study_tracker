# E2E Tests

Набор end-to-end сценариев в `e2e_tests/test_e2e_workflows.py`.

## Сценарии
1. Регистрация -> логин -> завершение Pomodoro -> достижение.
2. Stopwatch -> Pomodoro в одной пользовательской сессии.
3. Негатив: дубликат регистрации + продолжение работы существующего пользователя.
4. Негатив: ошибочный логин и последующий успешный вход.
5. Позитивный путь внешних API (quote/time/schedule).
6. Негативный путь внешних API (fallback при timeout).
7. Изоляция данных двух пользователей.
8. Формирование помесячного отчета сессий.
9. Безопасный stop без активной сессии.
10. Валидация токена и привязка сессии к пользователю.

## Запуск
```bash
python -m pytest e2e_tests -m e2e -v
```
